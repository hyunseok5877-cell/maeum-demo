# 📌 마음 (Maeum) — 프로젝트 결정 스냅샷

> **이 문서의 역할**: 프로젝트 전반의 "지금 이 순간 확정된 것"을 단일 진실 원천(Single Source of Truth)으로 기록.
>
> **읽는 순서**: AI 어시스턴트는 작업 시작 전 이 파일을 무조건 먼저 읽고, 이 파일과 모순되는 기존 문서 내용이 있으면 **본 파일을 따른다**.
>
> **롤백 규칙**: 이 파일 하단 "체크포인트" 목록에서 원하는 버전 번호(e.g. v0.3)를 지정하면, 그 상태로 전체 결정을 되돌린다.
>
> 최종 갱신: **2026-04-24** / 현재 버전: **v0.3**

## 💰 비즈니스 모델 (하이브리드 확정)

### 수익 구조
| 트랙 | 내용 | 비고 |
|---|---|---|
| **객단가 중개수수료** | 경험 건당 20~30% (평균 25%) | 메인 수익원 |
| **Gold 멤버십** | 연 100만원 / 우선예약·5~10% 할인·월 1회 시크릿 경험 | **Phase 2(6개월 차) 도입** |
| **Black 멤버십 (초대제)** | 연 500만원 / 전담 큐레이터·15% 할인·비공개 경험 | Phase 3(12개월 차) |
| **풀커스텀 큐레이션 설계비** | 건당 50~200만원 | 역제안 큐레이션 의뢰 시 |

### Phase 1 운영 규칙
- **런칭 후 6개월은 객단가만** 운영 — 멤버십 혜택 설계할 데이터 수집 기간
- DB·엔티티는 처음부터 멤버십 대응 설계 (스키마 변경 없이 Phase 2 전환 가능)

### 객단가 포지션
- 타겟 평균 **300만원/건**
- 월 1건만 성사돼도 손해 없음 (bootstrap 모델)

## 🤖 핵심 기능

1. **RAG 챗봇** — 경험 DB 검색 후 AI가 조합 추천
2. **경험 성향 테스트** (MBTI 스타일 6문항 → 16유형)
3. **역제안 큐레이션** — 고객이 조건 입력 → 큐레이터가 3안 커스텀 설계 → 이메일 전송
4. **관리자 경험 CRUD** — 사용자(오너)가 직접 경험·가격·벤더 등록
5. **구글맵 기반 지역 탐색**

## 🚨 불변 규칙 (Invariants)

> 아래 규칙은 예외 없이 모든 작업에 적용. 위반 시 즉시 롤백.

1. **"여행·투어·관광" 단어 금지** — UI·카피·챗봇·설문·문서 전체. 대체어: 경험·체험·세션·큐레이션·프로그램·라운지
2. **숙박·항공 결합 상품 금지** — 여행업 재분류 리스크
3. **가격·벤더 확정은 오너만** — AI는 제안·분석만, 실제 금액·계약 결정은 사용자
4. **프라이빗 셰프·소개팅 상품화 금지** — Phase 1 제외. 관련 DB 레코드 생성 금지
5. **멤버십 UI 런칭 시 비활성** — Phase 2 시점 오너 명시 승인 후 활성화
6. **모든 문서 변경은 `CHANGELOG.md`에 기록** — append-only, 삭제 금지
7. **RAG 챗봇 · TossPayments 결제 = 프로젝트 최후 순위** (2026-04-24 확정). 공급·경험 상세·카탈로그·성향테스트·큐레이션문의 등 핵심 플로우 전부 완성 이후 마지막에 구현
